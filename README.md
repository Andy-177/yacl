# Yet Another Config Language

**YACL**是一种轻量、易读的配置语言。 

它在保持 YAML 可读性的同时，借鉴了 TOML 的显式引号规则，并引入了独特的引用系统和多行字符串处理方式。

文件扩展名：`.yacl` 或 `.ycl`

## 快速概览

```yacl
# 基本键值对
姓名: '张三
年龄: 30
身高: 1.75
激活: true
角色: none

# 嵌套对象
人员:
    地址:
        城市: '北京
        邮编: 100000

# 列表
爱好:
    - '阅读
    - '游戏
    - '徒步

# 引用
当前城市: [人员].[地址].[城市]   # 结果为 "北京"
```

## 语法详解

### 键（Key）

- **所有键均为字符串** —— 无需为键添加引号。
- 键可以包含除 `:` 和 `[` / `]`（这些符号用于引用）之外的任意字符，也可以包含空格（建议避免特殊符号以保持清晰）。
- 键区分大小写。

### 值（Value）

#### 字符串

YACL 提供了 **四种** 字符串表示方式：

| 定界符 | 行为说明 |
|--------|----------|
| `'...` （单引号，**不闭合**） | 从开头的 `'` 开始，**直到行尾**的所有内容都被视为字符串字面量。**即使出现 `#` 也不会被当作注释**。 |
| `"...` （双引号，**不闭合**） | 与单引号类似，但 **`#` 会开始注释** —— 字符串在行内第一个 `#` 处结束（或到行尾）。 |
| `''' ... '''` （三个单引号） | 多行字符串。**所有内容**（包括 `#` 和换行）均为字面量。开头/结尾的换行会被修剪（见[多行块](#三引号块)）。 |
| `""" ... """` （三个双引号） | 多行字符串，但 **包含 `#` 的行会被当作注释**（即这些行不会出现在字符串值中）。其余换行保留。 |

**示例：**
```yacl
a: '这一整行都是字符串，即使有 # 和特殊字符
b: "只有这部分是字符串，# 之后的内容是注释（被忽略）
c: '''
    多行
    包含 # 号
    和换行
    '''
d: """
    这一行保留
    # 这一行是注释，被移除
    这一行也保留
    """
```

#### 数字

- 整数：`123`、`-456`
- 浮点数：`3.14`、`-0.001`

无需引号。

#### 布尔值

以下词元被识别为布尔值（**区分大小写**，请使用小写形式）：
- `true`、`false`
- `on`、`off`
- `enabled`、`disabled`

#### 空值

词元 `none` 表示空（null）值。

### 列表（数组）

- 列表以行首的 `-`（短横线）引入。
- 缩进决定嵌套层级。
- 列表项可以是任意类型的值（包括对象和嵌套列表）。

**示例：**
```yacl
水果:
    - '苹果
    - '香蕉
    - '橙子

矩阵:
    -
        - 1
        - 2
        - 3
    -
        - 4
        - 5
        - 6
```
`矩阵` 是一个包含两个列表的列表，每个内层列表含三个数字。

### 对象（映射）

- 对象由键值对组成，通过**缩进**嵌套（空格或制表符均可，但需保持一致）。
- 没有显式的结束标记，缩进范围决定作用域。

```yacl
配置:
    超时: 30
    重试次数: 3
    服务器:
        主机: 'localhost
        端口: 8080
```

### 引用

YACL 允许**引用**配置中的其他值。语法如下：

- `[键名]` 访问当前对象中的键。
- `.`（点号）用于链式访问。
- `[索引]` 访问列表元素（索引从 0 开始）。  
  **重要：** 当数字**不**带方括号时，它被视为列表索引；当数字在方括号内时，它被视为字符串键。

**示例：**
```yacl
人员:
    姓名: 'Alice
    年龄: 30
    爱好:
        - '阅读
        - '游戏

# 引用
人员姓名: [人员].[姓名]          # => "Alice"
第一个爱好: [人员].[爱好].0     # => "阅读" （裸数字表示索引）
第二个爱好: [人员].[爱好].[1]  # 也可用方括号，但这里 [爱好] 是列表，所以 [1] 也视为索引（如果键 "1" 不存在）
```

*注意：列表索引请使用裸数字（如 `.0`）；若使用 `[0]`，它会被当作字符串键 `"0"`，可能不存在于列表中。*

### 注释

- **单行注释：** `#` —— 从 `#` 到行尾的内容被忽略，**除非**它出现在单引号字符串（其中 `#` 是字面量）或三个单引号块内。
- **多行注释：** `### ... ###` —— 两组三个井号之间的所有文本被完全忽略，可跨多行，可出现在任何位置。

```yacl
# 这是注释
值: 42   # 行内注释

###
这是多行注释。
可以包含 # 等特殊字符。
以另一个 ### 结束
###

含井号的字符串: '这个 # 不是注释
```

### 多行块

#### 三引号块

三引号块有特定的换行修剪规则：

- 对于 `'''`（三个单引号）和 `"""`（三个双引号），定界符之间的内容**原样保留**，但：
  - 如果开始定界符后面紧跟换行符，则该换行符**被忽略**（开头不会多一个空行）。
  - 如果结束定界符前面有换行符，则该换行符**被忽略**（结尾不会多一个空行）。
- 因此，以下四种写法产生完全相同的值 `"aaa\nbbb"`：

```yacl
x: '''
aaa
bbb
'''

y: '''aaa
bbb
'''

z: '''
aaa
bbb'''

w: '''aaa
bbb'''
```

#### 块标量符号

块标量符号提供了更灵活的多行字符串控制方式，分为两种：`|`（保留注释）和 `||`（移除注释）。

##### 基本语法

```
键: |行数
文本行1
文本行2
文本行3
```

或

```
键: ||行数
文本行1
文本行2
文本行3
```

其中 `行数` 是一个正整数，指定要包含的**有效内容行数**（不包含注释行和空行）。

##### `|` —— 保留注释

使用 `|` 时，块内的**所有行**（包括包含 `#` 的行）都被视为字符串内容，注释不会被特殊处理。

```yacl
level1:
    level2:
        text: |3
            111
            222
            # 这是注释，但由于使用 |，所以这行被当作字符串内容
            333
```

结果：`text` 的值为 `"111\n222\n# 这是注释，但由于使用 |，所以这行被当作字符串内容\n333"`（如果指定 3 行，则取前 3 行有效内容）。

##### `||` —— 移除注释

使用 `||` 时，块内包含 `#` 的行会被**完全忽略**（不纳入行数计数），只有非注释行才会计入指定的行数。

```yacl
level1:
    level2:
        text: ||3
            111
            222
            # 这是注释，由于使用 || 所以不包含
            333
            # 这行也是注释，被忽略
            444
```

结果：`text` 的值为 `"111\n222\n333"`（取前 3 个非注释行）。

##### 详细规则

1. **行数指定**：`|` 或 `||` 后面紧跟一个正整数（如 `|3`、`||5`），表示要提取的有效行数。
2. **空行处理**：空行（仅包含空白字符的行）不纳入行数计数，但会**保留**在最终字符串中（出现在对应位置）。
3. **缩进修剪**：所有行的共同前导缩进会被自动移除（类似 YAML 的块标量）。
4. **行数不足**：如果有效行数少于指定的数量，则取所有有效行。
5. **结束标志**：块在达到指定行数后结束，或遇到缩进减少时结束（以先到者为准）。

##### 综合示例

```yacl
level1:
    level2:
        # 使用 ||，跳过注释行
        text: ||4
            第一行
            第二行
            
            第三行（上面有个空行）
            # 这行被忽略
            第四行
            # 这行也被忽略
            第五行（超出 4 行，不会被包含）

        # 使用 |，保留注释
        raw: |3
            第一行
            # 这是注释，被保留
            第二行
            # 这行也被保留（超出 3 行，不包含）
```

结果：
- `text` 的值为 `"第一行\n第二行\n\n第三行（上面有个空行）\n第四行"`（4 个非注释行，空行保留）
- `raw` 的值为 `"第一行\n# 这是注释，被保留\n第二行"`（3 行，包含注释行）

##### 嵌套示例

```yacl
文档:
    标题: '示例文档
    内容: ||5
        # 第一章
        第一章内容
        第一章继续
        
        # 第二章
        第二章内容
        # 第二章注释
        第二章继续
        第二章结束（超出 5 行，不包含）
    备注: "这是额外备注
```

结果：`内容` 包含 5 个非注释行，空行保留在对应位置。

## 示例

### 完整配置示例

```yacl
# 应用设置
应用:
    名称: 'MyApp
    版本: 1.0.0
    调试模式: enabled

数据库:
    主机: 'localhost
    端口: 5432
    凭证:
        用户: 'admin
        密码: 'secret
        # 密码: 'dummy   # 这行被注释

功能:
    - '身份验证
    - '日志
    - '分析

# 引用第一个功能
默认功能: [功能].0   # => "身份验证"

# 混合类型的嵌套列表
矩阵:
    -
        - 1
        - '二
        - 3.0
    -
        - 4
        - 5
        - '六

# 多行字符串，保留注释（单引号）
描述: '''
这个应用用于……
# 这一行属于字符串内容
以及这一行。
'''

# 多行字符串，去除注释行（双引号）
自述: """
# 概述
YACL 是……
# 安装
……
"""

# 使用 | 保留注释的块标量
日志配置: |5
    # 日志级别
    level: INFO
    # 输出格式
    format: json
    # 保留这些配置行
    
# 使用 || 移除注释的块标量
代码片段: ||3
    def hello():
        # 打印问候
        print("Hello")
        # 返回结果
        return True
```

## 注意事项与边界情况

- **缩进**：可使用空格或制表符，但须保持一致。YACL 使用缩进表示嵌套，与 YAML 类似。
- **引用中的裸数字**：不带方括号的数字始终作为列表索引；带方括号的数字（如 `[1]`）则作为字符串键 `"1"`。
- **布尔值词元**：区分大小写 —— `True` **不是**布尔值，只有 `true` 才是。
- **双引号字符串中的注释**：在单行双引号字符串（`"..."`）中，`#` 会终止字符串，因此字符串在 `#` 前结束。在三个双引号块中，包含 `#` 的整行会被移除。
- **空值**：仅 `none` 表示空。
- **引用不存在的键**：行为未定义，实现可返回 `none` 或抛出错误。
- **块标量行数**：`|` 和 `||` 后面的数字必须为正整数（≥1），否则视为语法错误。
- **块标量嵌套**：块标量内容不能包含比当前层级更浅的缩进（否则块会提前结束）。
