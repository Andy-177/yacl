# Yet Another Config Language

**YACL**是一种轻量、易读的配置语言。

它在保持 YAML 可读性的同时，借鉴了 TOML 的显式引号规则，并引入了独特的引用系统和多行字符串处理方式。

文件扩展名：`.yacl` 或 `.ycl`

## 快速概览

```yacl
# 基本键值对
姓名: '张三
年龄: 30
身高: 1.75
激活: true
角色: none

# 嵌套对象
人员:
    地址:
        城市: '北京
        邮编: 100000

# 列表
爱好:
    - '阅读
    - '游戏
    - '徒步

# 引用
当前城市: [人员].[地址].[城市]   # 结果为 "北京"
```

## 语法详解

### 键（Key）

- **所有键均为字符串** —— 无需为键添加引号。
- 键可以包含除 `:` 和 `[` / `]`（这些符号用于引用）之外的任意字符，也可以包含空格（建议避免特殊符号以保持清晰）。
- 键区分大小写。

### 值（Value）

#### 字符串

YACL 提供了 **四种** 字符串表示方式：

| 定界符 | 行为说明 |
|--------|----------|
| `'...` （单引号，**不闭合**） | 从开头的 `'` 开始，**直到行尾**的所有内容被视为字符串字面量。**即使出现 `#` 也不会被当作注释**。 |
| `"...` （双引号，**不闭合**） | 与单引号类似，但 **`#` 会开始注释** —— 字符串在行内第一个 `#` 处结束（或到行尾）。 |
| `''' ... '''` （三个单引号） | 多行字符串。**所有内容**（包括 `#` 和换行）均为字面量。开头/结尾的换行会被修剪（见[多行块](#三引号块)）。 |
| `""" ... """` （三个双引号） | 多行字符串，但 **包含 `#` 的行会被当作注释**（即这些行不会出现在字符串值中）。其余换行保留。 |

**示例：**
```yacl
a: '这一整行都是字符串，即使有 # 和特殊字符
b: "只有这部分是字符串，# 之后的内容是注释（被忽略）
c: '''
    多行
    包含 # 号
    和换行
    '''
d: """
    这一行保留
    # 这一行是注释，被移除
    这一行也保留
    """
```

#### 数字

- 整数：`123`、`-456`
- 浮点数：`3.14`、`-0.001`

无需引号。

#### 布尔值

以下词元被识别为布尔值（**区分大小写**，请使用小写形式）：
- `true`、`false`
- `on`、`off`
- `enabled`、`disabled`

#### 空值

词元 `none` 表示空（null）值。

### 列表（数组）

- 列表以行首的 `-`（短横线）引入。
- 缩进决定嵌套层级。
- 列表项可以是任意类型的值（包括对象和嵌套列表）。

**示例：**
```yacl
水果:
    - '苹果
    - '香蕉
    - '橙子

矩阵:
    -
        - 1
        - 2
        - 3
    -
        - 4
        - 5
        - 6
```
`矩阵` 是一个包含两个列表的列表，每个内层列表含三个数字。

### 对象（映射）

- 对象由键值对组成，通过**缩进**嵌套（空格或制表符均可，但需保持一致）。
- 没有显式的结束标记，缩进范围决定作用域。

```yacl
配置:
    超时: 30
    重试次数: 3
    服务器:
        主机: 'localhost
        端口: 8080
```

### 引用

YACL 允许**引用**配置中的其他值。语法如下：

- `[键名]` 访问当前对象中的键。
- `.`（点号）用于链式访问。
- `[索引]` 访问列表元素（索引从 0 开始）。  
  **重要：** 当数字**不**带方括号时，它被视为列表索引；当数字在方括号内时，它被视为字符串键。

**示例：**
```yacl
人员:
    姓名: 'Alice
    年龄: 30
    爱好:
        - '阅读
        - '游戏

# 引用
人员姓名: [人员].[姓名]          # => "Alice"
第一个爱好: [人员].[爱好].0     # => "阅读" （裸数字表示索引）
第二个爱好: [人员].[爱好].[1]  # 也可用方括号，但这里 [爱好] 是列表，所以 [1] 也视为索引（如果键 "1" 不存在）
```

*注意：列表索引请使用裸数字（如 `.0`）；若使用 `[0]`，它会被当作字符串键 `"0"`，可能不存在于列表中。*

### 注释

- **单行注释：** `#` —— 从 `#` 到行尾的内容被忽略，**除非**它出现在单引号字符串（其中 `#` 是字面量）或三个单引号块内。
- **多行注释：** `### ... ###` —— 两组三个井号之间的所有文本被完全忽略，可跨多行，可出现在任何位置。

```yacl
# 这是注释
值: 42   # 行内注释

###
这是多行注释。
可以包含 # 等特殊字符。
以另一个 ### 结束
###

含井号的字符串: '这个 # 不是注释
```

### 多行块

#### 三引号块

三引号块有特定的换行修剪规则：

- 对于 `'''`（三个单引号）和 `"""`（三个双引号），定界符之间的内容**原样保留**，但：
  - 如果开始定界符后面紧跟换行符，则该换行符**被忽略**（开头不会多一个空行）。
  - 如果结束定界符前面有换行符，则该换行符**被忽略**（结尾不会多一个空行）。
- 因此，以下四种写法产生完全相同的值 `"aaa\nbbb"`：

```yacl
x: '''
aaa
bbb
'''

y: '''aaa
bbb
'''

z: '''
aaa
bbb'''

w: '''aaa
bbb'''
```

#### 块标量符号

块标量符号提供了更灵活的多行字符串控制方式，分为两种模式：依赖缩进模式和带行数模式。

##### 基本语法

块标量有两种使用方式：

**1. 依赖缩进模式（不指定行数）**

```
键: |
    文本行1
    文本行2
    文本行3
```

或

```
键: ||
    文本行1
    文本行2
    文本行3
```

在这种模式下，块的内容由**缩进**决定——块从下一行开始，直到缩进回到当前层级为止。

**2. 带行数模式（指定行数）**

```
键: |3
文本行1
文本行2
文本行3
```

或

```
键: ||2
文本行1
文本行2
```

在这种模式下，`|` 或 `||` 后面紧跟一个正整数（如 `|3`、`||5`），表示要提取的行数。**块内容不需要缩进**，从下一行开始逐行读取。

##### `|` —— 保留注释

使用 `|` 时，块内的**所有行**（包括包含 `#` 的行）都被视为字符串内容，注释不会被特殊处理。

**依赖缩进模式示例：**

```yacl
level1:
    text1: |
        我是第一行文本
        我是第二行
        # 我是注释，但是由于是单个管道符（|），所以注释被保留
            我是第四行，但我还是属于text1中的多行文本
    tips: '我和text1键同级，所以我是独立的键值对，不属于多行文本
```

结果：`text1` 的值为 `"我是第一行文本\n我是第二行\n# 我是注释，但是由于是单个管道符（|），所以注释被保留\n    我是第四行，但我还是属于text1中的多行文本"`。

**带行数模式示例：**

```yacl
level2:
    text1: |3
我是第一行文本，由于有行数标注，所以不需要缩进
# 我是注释，但是由于是单个管道符（|），所以注释被保留
     我是第三行，由于有行数标注，空格会被保留下来
    text2: '我是与text1同级的键
```

结果：`text1` 的值为 `"我是第一行文本，由于有行数标注，所以不需要缩进\n# 我是注释，但是由于是单个管道符（|），所以注释被保留\n     我是第三行，由于有行数标注，空格会被保留下来"`（取前 3 行，缩进和注释均保留）。

##### `||` —— 移除注释

使用 `||` 时，包含 `#` 的行会被**完全忽略**（不纳入行数计数，也不出现在最终字符串中）。

**依赖缩进模式示例：**

```yacl
level1:
    text2: ||
        我是第一行文本
        # 我是注释，但是由于是双管道符（||），所以注释不会被保留
        key: '我看着是键值对，但是我在text2的缩进下面，所以我还是包含在多行文本里，不是键值对
    key: '我才是真正的键值对，我和text1、text2、tips键同级
```

结果：`text2` 的值为 `"我是第一行文本\nkey: '我看着是键值对，但是我在text2的缩进下面，所以我还是包含在多行文本里，不是键值对"`（注释行被移除，缩进范围内的所有非注释行均被包含）。

**带行数模式示例：**

```yacl
level2:
    text2: ||2
我是第一行文本
# 我是注释，但是由于是双管道符（||），所以注释不会被保留
我是第二行文本
# 这行也是注释，被忽略
```

结果：`text2` 的值为 `"我是第一行文本\n我是第二行文本"`（取前 2 个非注释行）。

##### 详细规则

**依赖缩进模式：**

1. **块起始**：`|` 或 `||` 后直接换行（无数字）。
2. **缩进决定范围**：从下一行开始，所有缩进大于当前键的行都属于该块。
3. **块结束**：当遇到缩进小于或等于当前键的行时，块结束。
4. **空行处理**：空行（仅包含空白字符的行）被保留在最终字符串中。
5. **缩进修剪**：所有行的共同前导缩进会被自动移除（类似 YAML 的块标量）。
6. **注释处理**：`||` 会移除包含 `#` 的行；`|` 则保留所有行。

**带行数模式：**

1. **行数指定**：`|` 或 `||` 后面紧跟一个正整数（如 `|3`、`||5`），表示要提取的行数。
2. **不需要缩进**：块内容从下一行开始逐行读取，不需要额外的缩进。
3. **行数计数**：
   - 对于 `|`：所有行（包括空行和注释行）都计入行数。
   - 对于 `||`：只有非注释行（不包含 `#` 的行）计入行数；注释行被完全忽略。
4. **空行处理**：空行（仅包含空白字符的行）不纳入行数计数，但会**保留**在最终字符串中（出现在对应位置）。
5. **块结束**：达到指定行数后块自动结束，或遇到缩进减少时结束（以先到者为准）。
6. **行数不足**：如果有效行数少于指定的数量，则取所有有效行。

##### 综合示例

```yacl
# 依赖缩进模式
level1:
    text1: |
        我是第一行文本
        我是第二行
        # 我是注释，但是由于是单个管道符（|），所以注释被保留
            我是第四行，但我还是属于text1中的多行文本
    tips: '我和text1键同级，所以我是独立的键值对，不属于多行文本
    text2: ||
        我是第一行文本
        # 我是注释，但是由于是双管道符（||），所以注释不会被保留
        key: '我看着是键值对，但是我在text2的缩进下面，所以我还是包含在多行文本里，不是键值对
    key: '我才是真正的键值对，我和text1、text2、tips键同级

# 带行数模式
level2:
    text1: |3
我是第一行文本，由于有行数标注，所以不需要缩进
# 我是注释，但是由于是单个管道符（|），所以注释被保留
     我是第三行，由于有行数标注，空格会被保留下来
    text2: ||2
```

```yacl

应用:
    名称: 'MyApp
    版本: 1.0.0
    调试模式: enabled

    主机: 'localhost
    端口: 5432
    凭证:
        用户: 'admin
        密码: 'secret
        # 密码: 'dummy   # 这行被注释

功能:
    - '身份验证
    - '日志
    - '分析

# 引用第一个功能
默认功能: [功能].0   # => "身份验证"

# 混合类型的嵌套列表
矩阵:
    -
        - 1
        - '二
        - 3.0
    -
        - 4
        - 5
        - '六

# 多行字符串，保留注释（单引号）
描述: '''
这个应用用于……
# 这一行属于字符串内容
以及这一行。
'''

# 多行字符串，去除注释行（双引号）
自述: """
# 概述
YACL 是……
# 安装
……
"""

# 使用 | 的依赖缩进模式，保留注释
日志配置: |
    # 日志级别
    level: INFO
    # 输出格式
    format: json
    保留这些配置行

# 使用 || 的依赖缩进模式，移除注释
代码片段: ||
    def hello():
        # 打印问候
        print("Hello")
        # 返回结果
        return True

# 使用 | 的带行数模式，保留注释
示例文本: |4
第一行
# 这是注释，被保留
第二行
第三行
# 这行超出指定行数，不包含
```

## 注意事项与边界情况

- **缩进**：可使用空格或制表符，但须保持一致。YACL 使用缩进表示嵌套，与 YAML 类似。在依赖缩进的块标量中，缩进决定了块的边界。
- **引用中的裸数字**：不带方括号的数字始终作为列表索引；带方括号的数字（如 `[1]`）则作为字符串键 `"1"`。
- **布尔值词元**：区分大小写 —— `True` **不是**布尔值，只有 `true` 才是。
- **双引号字符串中的注释**：在单行双引号字符串（`"..."`）中，`#` 会终止字符串，因此字符串在 `#` 前结束。在三个双引号块中，包含 `#` 的整行会被移除。
- **空值**：仅 `none` 表示空。
- **引用不存在的键**：行为未定义，实现可返回 `none` 或抛出错误。
- **块标量模式区分**：
  - `|` 或 `||` 后**直接换行**（无数字）→ 依赖缩进模式，缩进决定块范围。
  - `|` 或 `||` 后**紧跟正整数**（如 `|3`、`||2`）→ 带行数模式，块内容不需要缩进。
- **块标量注释处理**：
  - `|`（单管道）：保留所有行，注释行作为字符串内容。
  - `||`（双管道）：移除包含 `#` 的行。
- **块标量嵌套**：在依赖缩进模式中，块标量内容不能包含比当前层级更浅的缩进（否则块会提前结束）。在带行数模式中，块在达到指定行数后结束，或遇到缩进减少时结束（以先到者为准）。
